# Турнирный регламент и технические правила (Quake Live)

Шаблон **общих** технических правил и указатель на регламенты конкретных событий.

| Тип | Где лежит |
|-----|-----------|
| Общий шаблон (cvars, честная игра) | этот файл |
| Регламент конкретного турнира (SoT) | `ql-public-data/tournaments/{slug}/regulations.md` |
| Публичный HTML (generic) | [`player-guide/index.html`](../player-guide/index.html) |

**Пример события:** [Fast Learning Cup](tournaments/FAST-LEARNING-CUP.md) → `ql-public-data/tournaments/fast-learning-cup/regulations.md`

Источники фактов по cvars: `skills/ql-live` (runtime dumps), `ql-server-core/baseq3/server.cfg`.

## 1. Общие положения

- Турнир проводится на серверах Quake Live (QLDS), управляемых через ql-hub.
- Решение судьи/администратора по спорным ситуациям окончательно, если иное не указано в регламенте конкретного события.
- Игрок обязан использовать легальный клиент Quake Live (Steam). Goldberg / сторонние клиенты - только если явно разрешены регламентом (spec-боты - отдельный контур).

Для нового турнира создайте `ql-public-data/tournaments/{slug}/regulations.md` (см. `fast-learning-cup`).

## 2. Честная игра и настройки клиента

### 2.1. Что считается допустимым

- Стандартные cvars движка Quake Live в разрешённых диапазонах.
- Рекомендуемый сетевой профиль (см. раздел 3) - **не обязателен**, но **рекомендуется** для равных условий.

### 2.2. Запрещено

- Читы, инжекты, модификации клиента, дающие информацию или преимущество вне движка.
- Намеренное создание лишней нагрузки на сервер (флуд, спам командами).
- Использование cvars вне диапазона движка (обход лимитов).

### 2.3. `cl_timeNudge` - не чит, но есть нюансы

- `cl_timeNudge` и `cl_autoTimeNudge` - **легальные** клиентские cvars (в QL диапазон **-20 … 0**).
- Это настройка **отображения**: она влияет на интерполяцию моделей на вашем экране.
- Сервер **не получает** эти значения, и они **не меняют** серверный lag compensation и регистрацию попаданий.
- Рекомендация для турниров: **`cl_timeNudge 0`**, **`cl_autoTimeNudge 0`** (единые условия, без экспериментов).

## 3. Рекомендуемый сетевой профиль игрока

Базовые значения (турнир, стабильная сеть):

| Параметр | Значение |
|----------|-----------|
| `rate` | `25000` |
| `cl_maxpackets` | `125` |
| `cl_packetdup` | `0` |
| `cl_timeNudge` | `0` |
| `cl_autoTimeNudge` | `0` |

### Подстройка под сеть

| Ситуация | Что сделать |
|----------|-------------|
| Всё стабильно | Базовый профиль выше |
| Потеря пакетов (`cg_lagometer 1`, зелёные spikes) | `cl_packetdup 1`, при сильном loss - до `2-3` |
| Ping до ~60 ms | `cl_timeNudge 0`, `cl_autoTimeNudge 0` |
| Ping 70-100+ ms | Опционально `cl_timeNudge -10` … `-20` (только отображение на экране) |
| `cl_maxpackets` | `125` - стандарт; не снижать без явной причины |

`cl_timeNudge` не меняет серверный lag compensation и факт попаданий.
