Архитектура

Game VPS ──telemetry (10 Hz)──► ql-stats-hub ◄── WebSocket ── live-overlay (карта / табло)
              │
              └── ZMQ ──► ql-hub ──git push──► ql-public-data ──CDN──► stream-overlay (meta, /connect)

Карта: minqlx stream_telemetry → stats-hub /api/ingest/telemetry → WS /api/ws/live
Счёт:  ZMQ → hub → stats-hub /api/ingest/events
PNG + map_transforms.json: ql-stream-tools/live-overlay (jsDelivr или локальный HTTP)

Overlays — presentation layer (ql-stream-tools). Не нагружают game server. Hub UI копирует готовые URL после деплоя stats-hub (вкладка Статистика).

stream-overlay Tournament popup

Логотип турнира + popup live-матча для OBS. Chroma key фона popup: #00ff00.

OBS

  1. Browser Source → локальный файл stream-overlay/index.html (или CDN после push).
  2. 1920×1080, Refresh после смены настроек.
  3. Color Key / Chroma Key по зелёному фону карточки.

Не открывайте map через file:// для калибровки — браузер блокирует map_transforms.json. Используйте jsDelivr или локальный HTTP (см. ниже).

Калибровка карты

stats-hub отдаёт сырые x, y, z, yaw. Преобразование world → пиксели миникарты — в live-overlay/maps/map_transforms.json + PNG в live-overlay/maps/.

Debug UI (map.html?debug=1)

  1. Поднять статику: ql-local\overlay-serve.cmd → http://127.0.0.1:8787/
  2. Открыть http://127.0.0.1:8787/map.html?base=http://STATS_HOST:8090&debug=1
  3. Сетка (128/256/512 world units) — подогнать scale по ходьбе на сервере
  4. Scale — world width; Offset — центр мира на PNG; drag / wheel на карте
  5. Copy JSON entry → вставить в map_transforms.json, push ql-stream-tools

Поля map_transforms.json

ПолеСмысл
world_min_x … world_max_yДиапазон мировых координат на весь PNG
image_width / image_heightРазмер PNG в пикселях
image_urlПуть, напр. maps/bloodrun.png

Ключ = lowercase map_name из телеметрии. Placeholders: python scripts/gen_map_placeholders.py

Плавность точек на карте

Телеметрия ~10 Hz. По умолчанию включено EMA-сглаживание на requestAnimationFrame.

URLЭффект
smooth=1 (default)Плавное движение точки
smooth=0Резкий snap каждый пакет (~100 ms) — для сравнения
smooth_ms=280Медленнее догоняет цель, меньше дёрганий от шума

Hub и ql-local

  • Hub → Статистика — deploy stats-hub, Apply (телеметрия на QLDS), готовые overlay URL
  • ql-local\overlay-serve.cmd — локальный HTTP для live-overlay/ (порт 8787)
  • ql-local\overlay-stop.cmd / overlay-status.cmd

Телеметрия на VPS: qlx_streamTelemetryEnabled 1, URL/token задаёт hub Apply, не вручную в cfg.

Troubleshooting

СимптомПроверить
Нет точки на карте stats-hub /api/health → last_telemetry_by_server; Statistics → Apply; не file:// без assets
Точка не на миникарте Калибровка map_transforms.json; ?debug=1
CORS / Failed to fetch transforms HTTP или jsDelivr, не локальный file://
Дёргается при smooth=1 Увеличить smooth_ms; при debug=1 не двигать слайдеры во время проверки
Popup без счёта statsHubBase в stream-overlay; матч live в stats-hub
Пустой logo picker Hub publish в ql-public-data; верный publicDataBase
